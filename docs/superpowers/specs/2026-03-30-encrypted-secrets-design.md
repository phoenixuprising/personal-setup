# Encrypted Secrets Bundle

## Problem

`collect-secrets.py` gathers credentials into a plaintext `secrets/` directory that relies on physical USB security for transfer. This prevents storing the repo on GitHub since pushing secrets in plaintext is unsafe. Wallpapers (~34MB) are also mixed in with actual secrets, bloating the sensitive payload.

## Solution

Encrypt the secrets directory into a single `secrets.age` file using `age`, store the decryption key in 1Password, and commit the encrypted file to git. Separate non-secret user data (wallpapers) into its own collection script.

## Encryption & Key Management

- **Tool:** `age` (already installed, v1.3.1)
- **Key type:** X25519 keypair generated by `age-keygen`
  - Identity (private key): `AGE-SECRET-KEY-...` — stored in 1Password
  - Recipient (public key): `age1...` — used to encrypt, derivable from identity via `age-keygen -y`
- **1Password item:**
  - Type: Password
  - Title: `"system-ai Secrets Key"`
  - Password field: the age identity line
- **Key rotation:** each run of `collect-secrets.py` generates a fresh keypair and overwrites the 1Password item
- **Dependencies:** `age` and `op` CLI are hard requirements

## Script Changes

### `collect-secrets.py` (modified)

Collects only actual secrets and produces an encrypted bundle.

**Collected items (unchanged):**
- Claude Code credentials (`~/.claude/.credentials.json`)
- Git config (`~/.gitconfig`)
- SSH config and known_hosts (`~/.ssh/config`, `~/.ssh/known_hosts`)
- SSH public key from 1Password (for `authorized_keys`)
- 1Password account metadata

**New behavior:**
1. Fail early if `age` or `op` are not installed/signed-in
2. Collect secrets into `secrets/` as today
3. Generate fresh age keypair with `age-keygen`
4. Tar `secrets/` and encrypt with `age -r <recipient>` → `secrets.age`
5. Store identity in 1Password:
   - Check if `"system-ai Secrets Key"` exists with `op item get`
   - Use `op item create` if new, `op item edit` if existing
6. Remove plaintext `secrets/` directory
7. Print updated next-steps (push to GitHub instead of USB copy)

**Removed behavior:**
- Wallpaper collection (moved to `collect-userdata.py`)
- `.gitignore` management for `secrets/`

### `collect-userdata.py` (new)

Collects non-secret user data for USB transfer.

**Collected items:**
- Wallpapers from `~/Pictures/Wallpapers/` → `userdata/wallpapers/`

**Properties:**
- No encryption — nothing sensitive
- `userdata/` is gitignored (large binaries)
- Same color-coded logging pattern as `collect-secrets.py`
- Transferred via USB (not committed to git)

### `setup.py` step 6 (modified)

Decrypts and installs secrets in one step.

**Decrypt flow:**
1. If `secrets/` exists plaintext, use it directly (skip decryption)
2. If `secrets.age` exists:
   - Retrieve identity from 1Password: `op item get "system-ai Secrets Key" --fields password`
   - Write identity to temp file (mode 0600)
   - Decrypt: `age -d -i <temp_file> secrets.age` piped to `tar` extraction → `secrets/`
   - Clean up temp identity file
3. If neither exists, warn and skip

**Install flow (unchanged):**
- Claude credentials → `~/.claude/.credentials.json`
- Git config → `~/.gitconfig`
- SSH config, known_hosts, authorized_keys → `~/.ssh/`
- Configure sshd (pubkey-only)

**Wallpaper install (changed source):**
- Now reads from `userdata/wallpapers/` instead of `secrets/wallpapers/`

**Cleanup:**
- Remove extracted `secrets/` directory after install

### `.gitignore` changes

- `secrets/` — kept (transient plaintext during decrypt)
- `userdata/` — added (large binaries, USB transfer)
- `secrets.age` — tracked in git (encrypted, safe to push)
